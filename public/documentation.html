Welcome to the MoveX documentation

MoveX is a library of moves, that allows users to search for moves and create an account to safe their favorites.

Moves and user data is stored in a database (MongoDB), and accessed via a REST API.

Request URL HTTP Method Request body Response format Comment
Get list of all moves /moves GET none JSON (ID, names, links)
Get data about single move by name /moves/:name GET none JSON (move details)
Get data about a style /styles/:name GET none String + JSON (all moves in this style)
Get data about a source /sources/:name GET none JSON
Register new user /users POST JSON (name, email) JSON (user data)
Update user information /users/:name PUT string (newName) JSON (user data) At this stage only the username can be updated.
If the whole profile should be updated, the URL will be "/users" and the Request body "JSON".
Add move to favorite list /users/:name/favorites POST String (move ID) String
Remove move from favorite list /user/:name/favorites DELETE String (move ID) String
Deregister a user /users/:name DELETE none String
